# P5 SCALING PLAN: 50+ Corridors × 50+ Programs

**Objective:** Expand KEZA from 31 corridors + 21 programs to 50+ corridors + 50+ programs with zero performance regression.

**Duration:** 12–16 weeks (Phase 1–4)  
**Release date:** Q4 2026  
**Owner:** Engineering Team  
**Status:** Ready for autonomous execution

---

## EXECUTIVE SUMMARY

P5 is a coordinated expansion across **three dimensions:**

1. **Corridor Expansion** (31→50+ routes)
   - Add 6 new hubs + 40+ routes
   - Regions: Europe (LHR, FRA, AMS), Asia (HKG, BKK, KUL), Middle East (AUH/DOH), US (LAX, JFK, SFO, ORD, MIA), Africa (ADD, NBO, CMN, JNB)

2. **Loyalty Program Expansion** (21→50+ programs)
   - Add 30+ new programs
   - Coverage: 5 European carriers, 10 Asian programs, 5 Middle East/US Elite, 10 regional/transfer programs

3. **International Markets**
   - Latin America (MEX, SJO, GRU)
   - South Pacific (SYD, NZL)
   - Africa consolidation (JNB, ADD, KGL)

**Success Metrics:**
- ✓ All 50+ corridors <5s latency (p99) in Vercel production
- ✓ 50+ loyalty programs searchable and rankable
- ✓ Zero regression on existing B-tests / F-features
- ✓ Sentry error rate ≤0.1% for new routes
- ✓ Jest test suite >600 tests (all passing)

- Add automated scraping to detect chart changes

- Monitor p99 latency in Vercel analytics post-deploy
- Rollback if p99 >5s (restore previous CACHE_VERSION)

- Maintain homeCarrierGuarantee.test.ts to catch name mismatches
- Linter rule: all PROGRAM_TO_AIRLINE keys must exist in awardCharts.ts
- Pre-merge audit of all new program additions

**Mitigation:**
- Automated test: every corridor must fire at least 1 guarantee
- CI check: all corridors in ROUTE_AIRLINE_SUPPLEMENTS must have complementary HOME_CARRIER_PROGRAMS entry
- Manual audit of new routes pre-deploy
